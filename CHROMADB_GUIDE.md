# Vector Store — Location, Format & Seeding

> **Note:** This project previously used ChromaDB for vector storage. It was
> replaced with a pure-Python implementation (see `backend/app/services/vector_store.py`)
> because ChromaDB's index backend, `chroma-hnswlib`, has no prebuilt wheel
> for Python 3.13 on any platform — installing it requires a C++14 compiler
> (Visual Studio Build Tools on Windows), which many machines, especially
> locked-down corporate AVDs, don't have and can't easily install. The new
> implementation has **zero compiled dependencies** and needs nothing beyond
> what's already in `requirements.txt`. The public API (`upsert_requirement`,
> `query_similar_requirements`, `upsert_testcase`, `query_similar_testcases`,
> `upsert_knowledge_chunks`, `query_knowledge`, `delete_knowledge_doc`) is
> unchanged, so nothing elsewhere in the codebase needed to change.

## Where Vector Data Lives

By default, vector data is stored on disk as JSON files at:

```
smartqa_v21/chroma_data/
```

(The folder name is kept as `chroma_data` for continuity with earlier versions —
it has no other connection to ChromaDB.) This folder is created automatically
the first time the backend starts. It contains three files, one per collection:

| File                              | Used by | Purpose                                            |
|------------------------------------|---------|-----------------------------------------------------|
| `smartqa_requirements.json`        | Agent 1 | Past validated requirements (for "Similar Past Requirements" RAG) |
| `smartqa_testcases.json`           | Agent 2 | Past generated/approved test cases (for "Similar Test Cases" RAG) |
| `smartqa_knowledge.json`           | Both    | Uploaded BRD / Word / Confluence documents (knowledge base RAG)   |

Each file is a JSON object mapping `{id: {"embedding": [...], "document": "...", "metadata": {...}}}`.
Similarity search is done with a simple in-memory cosine-similarity scan over
all entries in a collection using numpy — entirely sufficient for a POC-scale
corpus (hundreds to low thousands of documents).

## How to Change the Location

Open `backend/.env` and set:

```
CHROMA_DATA_DIR=D:/SmartQA_VectorStore
```

Leave it blank to use the default `smartqa_v21/chroma_data` folder. This is useful if:
- You're migrating from TCS AVD to client AVD and want to keep the vector store on a
  specific drive
- You want multiple SmartQA installs to share one knowledge base (point them at the
  same network path)
- You need the data on a drive with more space

After changing this, **restart the backend** — the path is read once at startup.

## Seeding Sample Data (First-Time Setup)

On a fresh install, the `smartqa_testcases` collection is empty, so Agent 2's
"Similar Test Cases RAG" step returns nothing — you'll see "No similar test cases
in corpus yet." in the generation prompt. This is expected and not a bug.

To populate it with 8 sample SAP test cases (MM, SD, FI, HR modules) and 2 reference
knowledge documents (naming conventions, common T-codes), run:

```
seed_chroma.bat
```

(from the `smartqa_v21` root folder), or manually:

```
cd backend
venv\Scripts\activate
python scripts\seed_chroma.py
```

This is safe to re-run — it won't duplicate entries with the same ID.

## How Collections Get Populated Naturally

You don't need to seed manually forever — collections grow automatically as you use
the tool:

- **`smartqa_requirements`**: every time a requirement is validated by Agent 1, it's
  indexed automatically.
- **`smartqa_testcases`**: every time test cases are generated by Agent 2 (regardless
  of approval status), they're indexed automatically.
- **`smartqa_knowledge`**: every time you upload a document via the Knowledge Base
  panel in Agent 1, it's chunked and indexed automatically.

The seed script just gives you a head start so the first few demo runs already show
real RAG retrieval results instead of empty corpora.

## Inspecting What's Inside the Vector Store

Since data is now plain JSON, you can inspect it directly without any special tools:

```python
import json
with open("smartqa_v21/chroma_data/smartqa_testcases.json") as f:
    data = json.load(f)
print(len(data), "test cases indexed")
print(list(data.keys())[:5])  # first 5 IDs
```

Or even simpler, just open the `.json` file in any text editor or VS Code.

## Resetting / Clearing the Vector Store

To start completely fresh (e.g. testing or demo reset), stop the backend and delete
the folder:

```
rmdir /s /q smartqa_v21\chroma_data
```

It will be recreated empty on next backend startup.
